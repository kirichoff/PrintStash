Every collection can hold **documents** alongside its models: build notes, print
profiles, assembly instructions, spec sheets, datasheets — anything you'd
otherwise keep in a scattered folder. You can write Markdown directly in
PrintStash or upload PDFs and other files, and each collection can carry a
**README** shown at the top of its page.

:::note
Who can do what follows the **collection's role**. Anyone who can *view* a
collection can read its documents; *edit* or *admin* is required to create,
edit, or delete them.
:::

## The Documents tab

Open a collection and switch to the **Documents** tab. You'll see every document
attached to that collection, each tagged by kind (Markdown, PDF, or file) and
last-updated time. Returning from a document — via **Back** or the PrintStash
logo — brings you back to the Documents tab, not Models.

## Writing a Markdown document

Click **New document**. This drops you straight into the editor with an empty
draft.

- **Live preview** — toggle between **Edit** and **Preview** to see rendered
  Markdown.
- **Images** — paste from the clipboard or drag-and-drop image files into the
  editor; they're uploaded and embedded inline automatically.
- **Save** — a new document isn't actually created until you save. If you leave
  without saving, nothing is added.

## Uploading PDFs and files

Use **Upload** to add a PDF, Markdown file, plain-text file, or other document.

PDFs render **inline** in a themed viewer with page navigation, zoom, and
fit-to-width — no download required, and it matches the app's look instead of the
browser's default PDF chrome. Other file types offer a download.

## Collection READMEs

A collection can have a **README** that renders at the top of the collection
page — handy for a short description, links, or a table of contents for the
documents below. Edit it the same way as a Markdown document.
