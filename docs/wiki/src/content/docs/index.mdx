## Why I wanted this

Most print libraries start out tidy. Then a few months pass and you have
`hinge-final.gcode`, `hinge-final-fixed.gcode`, and a folder called `old maybe`.
The slicer remembers some settings, the printer remembers what is on its disk,
and your file browser remembers almost nothing useful.

PrintStash is the missing notebook for that mess. It keeps the model, the sliced
revisions, the thumbnail, the parser metadata, the printer copy, and your "this
one worked" note together. It is still a small self-hosted project, but it is
meant to be useful in a real home workflow today.

    indexes files **in place** — no copying, no second source of truth. See

  </Card>
  <Card title="Run it yourself" icon="laptop">

## Where your files live

You get to choose, per setup:

- **Vault** — PrintStash owns the bytes. Upload a file and it's copied into local
  disk or S3/R2-compatible storage. Simple, self-contained, easy to back up.
- **NAS folders (External Libraries)** — *your* folder stays the source of truth.
  PrintStash indexes files where they sit and only stores thumbnails and
  metadata, so the rest of your tooling keeps working unchanged. An unmounted
  share can never trigger a mass delete. See
  [NAS folders](/PrintStash/guides/nas-external-libraries/).

The two coexist: most people run a vault and link a NAS folder or two on top.
